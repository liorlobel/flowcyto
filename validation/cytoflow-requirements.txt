# Verified Python environment for the cytoflow cross-check (gen_golden_cytoflow.py).
# Captured 2026-06-10; Python 3.11 on macOS (Apple Silicon).
#
# WHY a plain `pip install -r` will NOT work: cytoflow 1.2 has no wheel for
# macOS arm64 / Python 3.11 (any version), and its sdist pins a 2021 build stack
# (old numpy) that fails to build under Python 3.11. Build cytoflow from source
# first (with build isolation disabled, so it uses the modern numpy/Cython present
# instead of its pinned ones), then install the rest as wheels:
#
#   python3.11 -m venv venv && source venv/bin/activate
#   pip install "numpy==1.26.4" "Cython==3.2.5" "swig==4.4.1"   # build prerequisites
#   pip install cytoflow==1.2 --no-build-isolation --no-deps     # compiles Moore's logicle C++ ext
#   pip install --no-deps -r validation/cytoflow-requirements.txt # --no-deps is REQUIRED: this file pins the
#       # full transitive closure, and cytoflow 1.2's metadata hard-pins numpy==1.21.2, so letting the resolver
#       # run would fail with ResolutionImpossible. --no-deps installs exactly the pinned set (verified clean).
#
# Run:
#   python validation/gen_golden_cytoflow.py validation/reference.fcs validation/golden.csv
#
# Pin rationale (cytoflow 1.2's 2021 deps lack py311 wheels, so each is the newest
# version that BOTH has a py311 wheel AND keeps cytoflow 1.2 importable):
#   numpy<2        ABI compatibility with numba 0.65 + the compiled logicle ext
#   seaborn 0.11.2 cytoflow imports the private seaborn.categorical._ViolinPlotter (gone in >=0.12)
#   traits 6.4.0   cytoflow uses traits.api.Function (removed in traits 7)
#   pandas 1.5.3 / matplotlib 3.6.3   compatible with seaborn 0.11.2
